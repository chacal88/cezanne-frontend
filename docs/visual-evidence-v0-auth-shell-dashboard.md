## Accepted deviations in this pass

- Dashboard card counts and calendar events remain technical/visual parity debt and are not accepted as final data truth.
- The seeded dashboard screenshot is accepted only as shell/layout evidence. It is not accepted as proof of live production authentication, live dashboard aggregates, or backend payload shape.
- Legacy dashboard evidence is limited because an authenticated legacy session was not captured in this pass.
- Legacy login bootstrap-failure and redirecting states were not captured in the focused same-run pass because the legacy flow immediately hands off after token set and needs broader authenticated app mocking; those states remain covered only by current deterministic hooks.
- Legacy `/session-lost` has no confirmed standalone matching route in this pass; the captured legacy route probe must not be treated as parity evidence for current `/session-lost`.
- Legacy `/confirm-registration` bootstrap-failure has no distinct source branch; the focused capture uses a mocked fallback probe and must not be treated as proof of a legacy-specific bootstrap-failure UI.
- Legacy register submitting capture uses a local loading probe because recaptcha/form-dealer blocks deterministic API submission in visual capture; it is coverage evidence, not replacement approval.
- Legacy `/users/invite-token` is not confirmed as a public invite acceptance route; the focused capture treats legacy `/users/invite-token` probes as nonmatching reference evidence only.
- Legacy Cezanne callback captures are resolver/redirect probes and must not be treated as proof of a route-local callback UI. Legacy SAML callback captures share the launch template and resolver/modal behavior; they remain reference evidence only until matched Figma/parity review.
- Notifications and inbox greenfield evidence is accepted as fixture-backed state coverage only, not as proof of live notification/inbox APIs.
- Explicit V0 decision: fixture-backed notification resolver visuals are accepted as temporary Figma input for destination categories because route ownership and resolver states are already contract-reviewed; live API replacement must preserve the same state categories.
- Explicit V0 decision: fixture-backed inbox list/detail visuals are accepted as temporary Figma input for conversation empty/selected states because live message transport remains outside V0; send/error/provider-blocked frames stay deferred.
