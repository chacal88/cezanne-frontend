# Frontend Greenfield ADR Index

## Purpose

This folder stores the durable architecture decisions for the greenfield frontend.

Use ADRs for:
- stable architectural patterns
- boundary rules
- provider abstractions
- routing/state/access decisions that should not be re-debated in feature PRs

## ADR format

Each ADR should stay short:
1. Context
2. Decision
3. Consequences
4. Alternatives rejected
